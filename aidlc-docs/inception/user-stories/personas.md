# User Personas - 테이블오더 서비스

## Overview
테이블오더 서비스의 주요 사용자 페르소나를 정의합니다. 각 페르소나는 시스템과 상호작용하는 방식, 목표, 동기, 특성을 포함합니다.

---

## Persona 1: 고객 (Customer)

### 기본 정보
- **이름**: 김민수
- **역할**: 매장 방문 고객
- **연령대**: 20-50대
- **기술 수준**: 기본적인 스마트폰/태블릿 사용 가능

### 목표 (Goals)
1. **빠른 주문**: 대기 시간 없이 즉시 메뉴를 주문하고 싶다
2. **편리한 탐색**: 메뉴를 쉽게 탐색하고 원하는 음식을 찾고 싶다
3. **주문 확인**: 내가 주문한 내역과 현재 상태를 확인하고 싶다
4. **간편한 사용**: 복잡한 로그인이나 회원가입 없이 바로 사용하고 싶다

### 동기 (Motivations)
- 직원을 기다리지 않고 자유롭게 주문하고 싶음
- 메뉴를 천천히 살펴보고 선택하고 싶음
- 주문 실수를 줄이고 정확하게 주문하고 싶음
- 주문 진행 상황을 실시간으로 알고 싶음

### 주요 특성 (Characteristics)
- **기술 친화도**: 중간 수준 - 기본적인 터치 인터페이스 사용 가능
- **사용 환경**: 매장 내 테이블에 비치된 태블릿
- **사용 빈도**: 매장 방문 시마다 (비정기적)
- **주요 관심사**: 빠른 주문, 메뉴 정보 확인, 주문 상태 확인

### 페인 포인트 (Pain Points)
- 직원을 호출하고 기다리는 시간이 불편함
- 메뉴판이 복잡하거나 정보가 부족할 때 선택이 어려움
- 주문이 제대로 전달되었는지, 언제 나올지 알 수 없어 불안함
- 추가 주문 시 다시 직원을 불러야 하는 번거로움

### 주요 사용 시나리오
1. **첫 방문 주문**
   - 테이블에 앉아 태블릿을 확인
   - 메뉴를 카테고리별로 탐색
   - 원하는 메뉴를 장바구니에 추가
   - 주문 확정 후 음식 대기

2. **추가 주문**
   - 태블릿에서 메뉴 화면 다시 열기
   - 추가 메뉴 선택 및 주문
   - 기존 주문과 새 주문 상태 확인

3. **주문 상태 확인**
   - 주문 내역 화면 접근
   - 각 주문의 현재 상태 확인 (대기중/준비중/완료)
   - 완료된 주문 확인 후 식사

## Persona 2: 매장 운영자 (Admin/Store Manager)

### 기본 정보
- **이름**: 박지영
- **역할**: 매장 관리자 / 운영자
- **연령대**: 30-50대
- **기술 수준**: 중급 - 업무용 소프트웨어 사용 경험 있음

### 목표 (Goals)
1. **효율적인 주문 관리**: 들어오는 주문을 실시간으로 확인하고 처리하고 싶다
2. **테이블 상태 파악**: 각 테이블의 주문 현황과 총 금액을 한눈에 보고 싶다
3. **메뉴 관리**: 메뉴를 쉽게 추가, 수정, 삭제하고 싶다
4. **운영 효율화**: 인건비를 절감하고 주문 실수를 줄이고 싶다
5. **매장 관리**: 여러 매장을 중앙에서 효율적으로 관리하고 싶다

### 동기 (Motivations)
- 주문 접수 및 전달 과정의 실수를 최소화하고 싶음
- 직원의 주문 접수 업무 부담을 줄이고 싶음
- 실시간으로 매장 운영 상황을 파악하고 싶음
- 메뉴 변경이나 가격 조정을 빠르게 반영하고 싶음
- 과거 주문 데이터를 분석하여 운영 개선에 활용하고 싶음

### 주요 특성 (Characteristics)
- **기술 친화도**: 중상급 - 업무용 시스템 사용에 익숙함
- **사용 환경**: 매장 내 관리자용 PC 또는 태블릿
- **사용 빈도**: 매일, 영업 시간 내내 (정기적, 지속적)
- **주요 관심사**: 주문 처리 속도, 테이블 회전율, 메뉴 관리, 매출 관리

### 페인 포인트 (Pain Points)
- 바쁜 시간대에 주문이 밀려 혼란스러움
- 직원의 주문 실수로 인한 고객 불만과 재작업
- 메뉴 변경 시 메뉴판 인쇄 비용과 시간 소요
- 테이블별 주문 현황을 파악하기 어려움
- 과거 주문 내역 조회 및 분석이 번거로움
- 여러 매장 운영 시 각 매장 상황 파악이 어려움

### 주요 사용 시나리오
1. **영업 시작 준비**
   - 시스템 로그인
   - 당일 메뉴 확인 및 조정
   - 테이블 상태 확인

2. **실시간 주문 관리**
   - 대시보드에서 신규 주문 실시간 확인
   - 주문 상태 변경 (대기중 → 준비중 → 완료)
   - 특정 주문 취소 또는 삭제 처리
   - 테이블별 총 주문액 확인

3. **테이블 세션 관리**
   - 고객 퇴장 시 테이블 세션 종료 처리
   - 과거 주문 내역으로 이동
   - 테이블 초기화 (다음 고객 대기)

4. **메뉴 관리**
   - 신규 메뉴 등록 (이름, 가격, 설명, 이미지)
   - 기존 메뉴 정보 수정
   - 품절 메뉴 삭제 또는 비활성화
   - 메뉴 노출 순서 조정

5. **과거 내역 조회**
   - 특정 테이블의 과거 주문 내역 조회
   - 날짜별 주문 내역 필터링
   - 매출 데이터 확인

6. **다중 매장 관리**
   - 여러 매장의 주문 현황 통합 확인
   - 매장별 메뉴 관리
   - 매장별 테이블 설정 및 관리

## Persona Mapping to System Features

### 고객 (Customer) 주요 기능
- 테이블 태블릿 자동 로그인
- 메뉴 조회 및 탐색
- 장바구니 관리
- 주문 생성
- 주문 내역 조회 (실시간 상태 업데이트)

### 관리자 (Admin) 주요 기능
- 매장 인증 및 로그인
- 실시간 주문 모니터링
- 주문 상태 변경
- 테이블 관리 (초기 설정, 세션 관리, 주문 삭제)
- 메뉴 관리 (CRUD)
- 과거 주문 내역 조회
- 다중 매장 관리

## Summary

두 페르소나는 테이블오더 서비스의 핵심 사용자 그룹을 대표합니다:

- **고객 (Customer)**: 빠르고 편리한 주문 경험을 원하는 매장 방문자
- **관리자 (Admin)**: 효율적인 주문 관리와 매장 운영을 원하는 매장 운영자

각 페르소나의 목표와 페인 포인트를 이해함으로써, 사용자 중심의 기능을 설계하고 구현할 수 있습니다.

## 향후 확장 가능한 페르소나 (Future Personas)

현재 MVP 범위에서는 제외되지만, 향후 시스템 확장 시 고려할 수 있는 추가 페르소나:

### Persona 3: 주방 직원 (Kitchen Staff) - 향후 고려
- **역할**: 주문을 받아 조리하는 주방 담당자
- **주요 목표**: 주문을 명확히 확인하고 조리 상태를 업데이트
- **주요 기능**: 주방 디스플레이 시스템, 조리 시작/완료 알림
- **MVP 제외 이유**: 현재는 관리자가 주문 상태를 직접 변경

### Persona 4: 서빙 직원 (Server) - 향후 고려
- **역할**: 완료된 음식을 테이블로 서빙하는 직원
- **주요 목표**: 완료된 주문을 확인하고 서빙 완료 처리
- **주요 기능**: 서빙 대기 목록, 서빙 완료 체크
- **MVP 제외 이유**: 현재는 관리자가 모든 주문 관리 담당

**시스템 설계 시 고려사항**: 역할 기반 접근 제어(RBAC)를 고려하여 향후 다양한 직원 역할을 쉽게 추가할 수 있도록 설계
