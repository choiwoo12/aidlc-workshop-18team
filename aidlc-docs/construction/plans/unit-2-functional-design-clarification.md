## Clarification Questions

### CQ1: 주문 생성 유효성 검증 (Q4 관련)

**Original Answer**: A (최소 검증 - 장바구니 비어있지 않음만)

**Risk Identified**: 메뉴 판매 불가 상태나 가격 변경 시 주문 생성 후 에러 발생 가능

**Clarification**: 최소 검증만 수행할 경우 다음 시나리오에서 문제가 발생할 수 있습니다:
- 시나리오 1: 고객이 장바구니에 메뉴를 담은 후, 관리자가 해당 메뉴를 판매 불가로 변경
- 시나리오 2: 고객이 장바구니에 메뉴를 담은 후, 관리자가 메뉴 가격을 변경

이러한 시나리오를 어떻게 처리하시겠습니까?

**A)** 최소 검증 유지 (주문 생성 후 서버에서 에러 반환)
- 장점: 빠른 주문 생성
- 단점: 주문 실패 시 사용자 불편
- 영향: 주문 실패 시 에러 메시지 표시 후 장바구니 유지

**B)** 기본 검증으로 변경 (주문 생성 전 메뉴 판매 가능 여부 확인)
- 장점: 주문 실패 가능성 감소
- 단점: 추가 API 호출 필요
- 영향: 주문 생성 시간 약간 증가 (0.5초 이내)

**C)** 상세 검증으로 변경 (메뉴 판매 가능 + 가격 일치 확인)
- 장점: 높은 안전성
- 단점: 추가 검증 시간
- 영향: 주문 생성 시간 증가 (1초 이내)

[Answer]: A

### CQ2: 주문 번호 순차 범위 (Q8 관련)

**Original Answer**: D (테이블 번호 + 순차 번호, 예: T01-001)

**Ambiguity Identified**: 순차 번호가 테이블별로 관리되는지, 매장 전체에서 관리되는지 불명확

**Clarification**: 주문 번호 "T01-001"에서 순차 번호 "001"의 범위를 명확히 해주세요.

**A)** 테이블별 순차 (T01-001, T01-002, T02-001, T02-002)
- 장점: 테이블별 주문 추적 용이
- 단점: 테이블별 카운터 관리 필요
- 예시: 테이블 1의 첫 주문 = T01-001, 테이블 2의 첫 주문 = T02-001

**B)** 매장 전체 순차 (T01-001, T02-002, T01-003, T03-004)
- 장점: 단순한 카운터 관리
- 단점: 테이블 번호와 순차 번호 불일치
- 예시: 테이블 1의 첫 주문 = T01-001, 테이블 2의 첫 주문 = T02-002

**C)** 날짜별 + 매장 전체 순차 (T01-20260209-001)
- 장점: 날짜 정보 포함, 매일 리셋
- 단점: 긴 주문 번호
- 예시: 2026-02-09 테이블 1의 첫 주문 = T01-20260209-001

[Answer]: A

### CQ3: 에러 메시지 상세도 (Q10 관련)

**Original Answer**: A (기본 에러 메시지만 표시)

**Risk Identified**: 네트워크 에러와 서버 에러를 구분하지 않아 사용자가 문제 원인을 파악하기 어려움

**Clarification**: 기본 에러 메시지만 표시할 경우 다음 시나리오에서 사용자 혼란이 발생할 수 있습니다:
- 시나리오 1: 네트워크 연결 끊김 (Wi-Fi 문제)
- 시나리오 2: 서버 에러 (500 Internal Server Error)
- 시나리오 3: 유효성 검증 실패 (장바구니 비어있음)

이러한 시나리오를 어떻게 구분하여 표시하시겠습니까?

**A)** 기본 메시지 유지 (모든 에러에 동일한 메시지)
- 장점: 단순함
- 단점: 사용자가 문제 원인 파악 어려움
- 예시: "주문 생성에 실패했습니다. 다시 시도해주세요."

**B)** 에러 타입별 메시지 (네트워크, 서버, 유효성 구분)
- 장점: 명확한 피드백
- 단점: 에러 분류 로직 필요
- 예시: 
  - 네트워크: "네트워크 연결을 확인해주세요."
  - 서버: "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요."
  - 유효성: "장바구니가 비어있습니다."

**C)** 에러 타입별 메시지 + 재시도 버튼 (네트워크 에러만)
- 장점: 사용자 편의성 향상
- 단점: 재시도 로직 필요
- 예시: 네트워크 에러 시 "재시도" 버튼 표시

[Answer]: A
